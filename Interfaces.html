Understanding Abstract Class and Interface in Java

What is an Abstract Class?

An abstract class in Java is a class that cannot be instantiated directly. It may contain both abstract methods (methods without a body) and concrete methods (methods with a body). Abstract classes allow you to provide a common base with some shared implementation for subclasses.

What is an Interface?

An interface defines a contract for classes that implement it. It cannot contain any method implementation (except default or static methods in recent versions of Java). A class can implement multiple interfaces, allowing Java to support multiple inheritance of behavior.

Java Code Example: Abstract Class

                
abstract class Animal {
    // Abstract method (no implementation)
    abstract void sound();

    // Concrete method
    void eat() {
        System.out.println("This animal eats food.");
    }
}

class Dog extends Animal {
    // Providing implementation of the abstract method
    void sound() {
        System.out.println("The dog barks.");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.sound();  // Output: The dog barks.
        dog.eat();    // Output: This animal eats food.
    }
}

Java Code Example: Interface

                
interface Animal {
    // Abstract method (no implementation)
    void sound();

    // Default method (has implementation)
    default void eat() {
        System.out.println("This animal eats food.");
    }
}

class Dog implements Animal {
    // Implementing the abstract method
    public void sound() {
        System.out.println("The dog barks.");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.sound();  // Output: The dog barks.
        dog.eat();    // Output: This animal eats food.
    }
}

Explanation of the Code

Abstract Class Example: In this example, we have an abstract class `Animal` with an abstract method `sound()` and a concrete method `eat()`. The class `Dog` extends `Animal` and provides an implementation for the abstract method `sound()`. The method `eat()` is inherited directly without any changes.

Interface Example: The interface `Animal` defines an abstract method `sound()` and a default method `eat()`. The class `Dog` implements the `Animal` interface, providing an implementation for the `sound()` method. The `eat()` method is inherited as a default method from the interface.

Key Differences:

  1. Abstract Class: Can have both abstract and concrete methods. Can have instance fields.
  2. Interface: Can only have abstract methods (except default/static methods). Cannot have instance fields.
  3. Inheritance: A class can extend only one abstract class but can implement multiple interfaces.
