Static and Final in Java

Static: The static keyword in Java is used for memory management and allows fields, methods, or blocks to belong to the class rather than any specific object.

  • Use case: Shared variables or utility methods, such as Math.sqrt().
  • Example:
public class StaticExample { static int counter = 0; // Shared across all instances public StaticExample() { counter++; } public static void showCounter() { System.out.println("Counter: " + counter); } public static void main(String[] args) { StaticExample obj1 = new StaticExample(); StaticExample obj2 = new StaticExample(); StaticExample.showCounter(); // Accessing static method } }

Final: The final keyword in Java is used to declare constants, prevent method overriding, or inheritance of a class.

  • Use case: Declaring constants, ensuring methods are not overridden, or restricting class inheritance.
  • Example:
public class FinalExample { final int MAX_VALUE = 100; // Constant public void displayValue() { System.out.println("Max Value: " + MAX_VALUE); } public static void main(String[] args) { FinalExample example = new FinalExample(); example.displayValue(); } } final class ImmutableClass { // This class cannot be extended }
Note:
  • Static members belong to the class and are shared among all instances.
  • Final variables cannot be reassigned, final methods cannot be overridden, and final classes cannot be extended.
