Class and Object in Java

Learn about classes, objects, and references in Java programming.

What is a Class?

A class in Java is a blueprint for creating objects. It defines a datatype by bundling data and methods that work on the data into one single unit.

In other words, a class defines properties (fields) and behaviors (methods) that the objects created from it will have.

What is an Object?

An object is an instance of a class. When a class is defined, no memory is allocated until an object of that class is created.

Each object has its own copy of instance variables and can use the methods defined in the class.

What is a Reference?

A reference in Java is a variable that holds the memory address of an object. It does not store the actual object, but rather points to it in memory. The reference variable allows you to interact with the object and access its methods and attributes.

Code Example: Class, Object, and Reference


public class Car {
    // Fields (Attributes)
    String make;
    String model;
    int year;

    // Constructor to initialize the object
    public Car(String make, String model, int year) {
        this.make = make;
        this.model = model;
        this.year = year;
    }

    // Method to display the details of the car
    public void displayInfo() {
        System.out.println("Car Make: " + make);
        System.out.println("Car Model: " + model);
        System.out.println("Car Year: " + year);
    }
}

public class Main {
    public static void main(String[] args) {
        // Reference variable of type Car
        Car myCar;

        // Creating a new Car object and assigning it to the reference variable
        myCar = new Car("Honda", "Civic", 2022);

        // Using the reference variable to access the object's method
        myCar.displayInfo();  // Output: Car Make: Honda, Car Model: Civic, Car Year: 2022
    }
}

Explanation of the Code

  • Reference Variable: The variable myCar is a reference variable of type Car. It holds the memory address of the Car object.
  • Creating an Object: We use the new keyword to create a new object of the Car class and assign it to the reference variable myCar.
  • Accessing Object Methods: The reference variable myCar is used to call the displayInfo() method, which displays the car's make, model, and year on the console.

Key Points About References:

  • Reference Variables: A reference variable points to an object stored in memory. It does not hold the actual object but its memory address.
  • Null Reference: A reference variable that does not point to any object is said to be null. Accessing methods or fields from a null reference will result in a NullPointerException.
  • Objects: Objects are instances of classes created with the new keyword and stored in memory, while references are used to interact with these objects.

Additional Notes on References:

  • Reference Type: A reference type is any type that refers to an object (such as classes, interfaces, arrays, etc.), unlike primitive types (int, float, etc.), which store the actual value.
  • Passing by Reference: In Java, when you pass an object to a method, you're passing the reference to that object. This means that changes made to the object inside the method will affect the original object.
