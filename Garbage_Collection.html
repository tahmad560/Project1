Understanding Garbage Collection in Java

A comprehensive guide to understanding garbage collection and memory management in Java.

What is Garbage Collection?

Garbage collection in Java is a process by which the Java Virtual Machine (JVM) automatically reclaims memory by removing objects that are no longer in use or are unreachable. This helps in managing memory and ensuring efficient program execution.

How Does Garbage Collection Work?

When objects are created in Java, they consume memory. However, when they are no longer needed (i.e., they become unreachable), the JVM can automatically reclaim their memory. This is done by the garbage collector, which tracks objects' references and frees up memory when objects are no longer reachable.

Ways Objects Become Eligible for Garbage Collection:

  • Unreachable Objects: Objects that no longer have any active references pointing to them become unreachable and are eligible for garbage collection.
  • Setting References to Null: By setting a reference to null, the object becomes eligible for garbage collection if no other references point to it.
  • Reassigning References: If a reference is reassigned to another object, the previous object becomes unreachable and may be garbage collected.

Example 1: Unreachable Object

This example demonstrates how an object becomes eligible for garbage collection when it is no longer referenced.

class GarbageCollectionExample {
    public static void main(String[] args) {
        MyClass obj1 = new MyClass(); // Object 1 is created
        MyClass obj2 = obj1; // obj2 refers to the same object as obj1
        
        obj1 = null; // obj1 no longer refers to the object, making it eligible for garbage collection
    }
}

class MyClass {
    MyClass() {
        System.out.println("Object created");
    }
}

In this example, when obj1 is set to null, the object it referred to becomes eligible for garbage collection, as obj2 is the only reference left to the object.

Example 2: Reassigning References

Here, we demonstrate how reassigning references causes the old object to become eligible for garbage collection.

class GarbageCollectionExample {
    public static void main(String[] args) {
        MyClass obj1 = new MyClass(); // Object 1 is created
        MyClass obj2 = new MyClass(); // Object 2 is created
        
        obj2 = obj1; // obj2 now refers to the same object as obj1
        obj1 = new MyClass(); // obj1 now refers to a new object, making the old object eligible for garbage collection
    }
}

In this example, when obj1 is reassigned, the original object that obj1 and obj2 referred to becomes unreachable and eligible for garbage collection.

Example 3: Setting Reference to Null

This example shows how setting a reference to null makes the object eligible for garbage collection.

class GarbageCollectionExample {
    public static void main(String[] args) {
        MyClass obj1 = new MyClass(); // Object 1 is created
        obj1 = null; // obj1 is set to null, making the object eligible for garbage collection
    }
}

By setting obj1 to null, the object it referenced becomes eligible for garbage collection, as there are no remaining references to it.
