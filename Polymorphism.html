Understanding Polymorphism and instanceof in Java

What is Polymorphism?

Polymorphism is a core concept in Object-Oriented Programming (OOP) that allows an object to take on many forms. In Java, polymorphism is typically achieved through method overriding and method overloading:

  • Method Overriding: When a subclass provides a specific implementation of a method already defined in its superclass.
  • Method Overloading: When multiple methods with the same name exist, but with different parameter lists.
  • Subclass Polymorphism: When a subclass object is referred to by a superclass reference variable, allowing the overridden method in the subclass to be called at runtime.
  • instanceof Operator: Used to check if an object is an instance of a specific class or subclass.

Java Code Example: Polymorphism with instanceof

                
class Animal {
    // Method in the superclass
    public void sound() {
        System.out.println("The animal makes a sound.");
    }
}

class Dog extends Animal {
    // Method overriding in the subclass
    @Override
    public void sound() {
        System.out.println("The dog barks.");
    }

    // Method overloading in the subclass
    public void sound(int times) {
        for (int i = 0; i < times; i++) {
            System.out.println("Bark!");
        }
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Dog();  // Dog object is assigned to Animal reference
        myAnimal.sound();  // Output: The dog barks. (Subclass method is called)

        // Using instanceof to check the actual type of myAnimal
        if (myAnimal instanceof Dog) {
            System.out.println("myAnimal is an instance of Dog.");
        }

        // Method overloading example
        Dog myDog = new Dog();
        myDog.sound(3);  // Output: Bark! Bark! Bark!
    }
}

Expected Output:

The dog barks.

myAnimal is an instance of Dog.

Bark! Bark! Bark!

Explanation of the Code

Polymorphism: In this example, we demonstrate polymorphism through method overriding, method overloading, and subclass polymorphism.

Method Overriding: The method sound() in the Dog class overrides the same method in the Animal class. When the method is called on an object of type Dog, it prints "The dog barks." This is an example of method overriding where the subclass provides a specific implementation of the inherited method.

Method Overloading: The sound() method in the Dog class is overloaded. The method accepts an integer argument, which specifies how many times the dog will bark. If you call myDog.sound(3), it will print "Bark!" three times. This is an example of method overloading, where the same method name is used with different parameter lists.

Subclass Polymorphism: In this case, the variable myAnimal is of type Animal, but it references an object of type Dog. This demonstrates polymorphism, as the correct sound() method (from Dog) is called at runtime.

instanceof: The instanceof operator is used to check whether the reference variable myAnimal actually refers to an object of type Dog. The expression myAnimal instanceof Dog returns true because the actual object is of type Dog. This check helps us to safely confirm the type of the object before performing any specific operations.

Output Breakdown: When the program runs:

  1. The sound() method in the Dog class is called via the Animal reference. It prints "The dog barks."
  2. The instanceof check confirms that myAnimal is indeed a Dog object and prints "myAnimal is an instance of Dog."
  3. The overloaded sound(int times) method in the Dog class is called with a parameter of 3, and it prints "Bark!" three times.
