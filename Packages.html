Java Nested Packages Explained

Learn how to create and use nested packages in Java

What are Nested Packages?

Nested packages in Java are a hierarchical way of organizing code into subpackages. For example: package1.package2 represents a package package2 inside the parent package package1.

Why Use Nested Packages?

  • Better Code Organization: Helps in structuring large-scale projects logically.
  • Namespace Management: Avoids conflicts by creating unique namespaces.
  • Logical Grouping: Groups related classes and subpackages together.

Creating Nested Packages

To create a nested package, use dot notation in the package statement. The directory structure must match the hierarchy.

Example:

// File: package1/package2/MyClass.java
package package1.package2;

public class MyClass {
    public void display() {
        System.out.println("Hello from package1.package2!");
    }
}

Using Nested Packages

To use a class from a nested package, import it using its fully qualified name:

// File: Main.java
import package1.package2.MyClass;

public class Main {
    public static void main(String[] args) {
        MyClass obj = new MyClass();
        obj.display();
    }
}

Extending Nested Packages

You can extend classes from nested packages in another package. Here's an example:

// File: package3/ExtendedClass.java
package package3;

import package1.package2.MyClass;

public class ExtendedClass extends MyClass {
    public void extendedDisplay() {
        System.out.println("Extended functionality from package3!");
    }
}

Now, you can use the extended class in your main program:

// File: Main.java
import package3.ExtendedClass;

public class Main {
    public static void main(String[] args) {
        ExtendedClass obj = new ExtendedClass();
        obj.display(); // From package1.package2
        obj.extendedDisplay(); // From package3
    }
}
